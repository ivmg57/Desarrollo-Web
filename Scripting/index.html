Valores en JavaScript

Números

En JavaScript, los valores numéricos son de tipo `number`. Este tipo incluye tanto números enteros como decimales. Por ejemplo, el número 13 es de tipo `number`.

Operadores Aritméticos

JavaScript permite realizar operaciones aritméticas usando operadores como `+`, `-`, `*`, y `/`. La precedencia de estos operadores determina el orden en que se realizan las operaciones en una expresión.

100 + 4 * 11 = 144

En esta expresión, la multiplicación se realiza antes que la suma debido a la precedencia de los operadores. Los operadores `/` y `*` tienen la misma precedencia, al igual que `+` y `-`. Cuando hay múltiples operadores con la misma precedencia, se aplican de izquierda a derecha.

1 - 2 + 1 = 0 // (1 - 2) + 1

Strings (Cadenas de Texto)

Las cadenas de texto, o strings, se utilizan para representar texto y se escriben encerrándolas entre comillas simples (`'`), dobles (`"`), o backticks (```).

'Down on the sea'
"Lie on the ocean"
`Float on the ocean`

Operaciones con Strings

A diferencia de los números, las cadenas no pueden ser divididas, multiplicadas o restadas. Sin embargo, se pueden concatenar usando el operador `+`.

"con" + "cat" + "e" + "nate" = "concatenate"

Las cadenas delimitadas por backticks, conocidas como literales de plantilla (template literals), permiten incluir expresiones dentro de la cadena usando `${}`.

`half of 100 is ${100 / 2}` // "half of 100 is 50"

Valores Booleanos

Los valores booleanos representan dos estados posibles: `true` (verdadero) o `false` (falso). Estos son útiles para realizar comparaciones y tomar decisiones en el código.

3 > 2  // true
3 < 2  // false
"Garnet" != "Ruby"  // true
"Pearl" == "Amethyst"  // false

Operadores Lógicos

JavaScript soporta tres operadores lógicos principales: `&&` (AND), `||` (OR), y `!` (NOT). Estos operadores permiten realizar operaciones lógicas sobre valores booleanos.

AND (&&)

El operador `&&` produce `true` solo si ambos operandos son `true`.

true && false  // false
true && true  // true

OR (||)

El operador `||` produce `true` si al menos uno de los operandos es `true`.

false || true  // true
false || false  // false

NOT (!)

El operador `!` invierte el valor booleano de su operando.

!true  // false
!false  // true

Operador Ternario (?:)

El operador ternario selecciona uno de dos valores basándose en una condición.

true ? 1 : 2  // 1
false ? 1 : 2  // 2

Cortocircuito de Operadores Lógicos

Los operadores lógicos `&&` y `||` pueden devolver el valor original del operando dependiendo del resultado de la conversión a booleano.

null || "user"  // "user"
"Agnes" || "user"  // "Agnes"

Valores como `0`, `NaN`, y la cadena vacía `""` se consideran `false`, mientras que otros valores se consideran `true`.

Operador de Fusión Nula (??)

El operador `??` devuelve el valor de la derecha solo si el valor de la izquierda es `null` o `undefined`.

0 || 100  // 100
0 ?? 100  // 0
null ?? 100  // 100

Funciones

Las funciones son bloques de código que realizan una tarea específica. Se pueden definir usando la palabra clave `function` y se pueden llamar por su nombre.

const square = function(x) {
  return x * x;
};

console.log(square(12));  // 144

Las funciones ayudan a estructurar programas grandes, reducir la repetición, y asociar nombres con subprogramas.

Expresiones Regulares (REGEX)

Las expresiones regulares son patrones utilizados para buscar coincidencias dentro de cadenas de texto. Un ejemplo de REGEX en JavaScript es `/(\\+|-\\x||\\/)/`, que coincide con los caracteres `+`, `-`, `x`, o `/`.

Explicación de la REGEX:

  • `/`: Inicio de la expresión regular.
  • `()`: Agrupamiento de opciones.
  • `\\+`, `\\-`, `\\x`, `\\/`: Caracteres a coincidir, escapados para evitar su significado especial.
  • `|`: Operador "o", indicando opciones posibles.
  • `/`: Final de la expresión regular.

Declaración Switch

La declaración `switch` evalúa una expresión y ejecuta el bloque de código correspondiente al caso que coincida con el valor de la expresión. Si no hay coincidencia, se ejecuta el bloque `default`.

switch(expression) {
  case value1:
    // código a ejecutar
    break;
  case value2:
    // código a ejecutar
    break;
  default:
    // código a ejecutar si no hay coincidencia
}

HTML y CSS

HTML (HyperText Markup Language) y CSS (Cascading Style Sheets) son los pilares fundamentales del desarrollo web. HTML es el lenguaje de marcado que estructura el contenido de las páginas web, mientras que CSS es el lenguaje que se utiliza para describir su presentación. Esta presentación proporciona una visión detallada de cómo funcionan HTML y CSS, y cómo se utilizan para crear y diseñar páginas web.

HTML: Estructura y Marcado de Páginas Web

¿Qué es HTML?

HTML (HyperText Markup Language) permite crear enlaces (hipervínculos) que permiten a los visitantes navegar de una página a otra de manera rápida y fácil. Una lengua de marcado permite anotar texto, y estas anotaciones proporcionan significado adicional al contenido de un documento. Al agregar código alrededor del texto original que queremos mostrar, el navegador utiliza este código para mostrar la página correctamente. Estos códigos son las etiquetas de marcado (tags).

Cómo Utilizan las Páginas la Estructura

La estructura es crucial en cualquier documento, ya que ayuda a los lectores a entender el mensaje y a navegar por el contenido. En las páginas web, esto se logra mediante el uso de diferentes etiquetas HTML que organizan y estructuran el contenido de manera lógica y coherente.

Ejemplo de estructura de un artículo:

  • Título (headline)
  • Texto (body)
  • Imágenes (images)
  • Subtítulos (subheadings)
  • Citas (quotes)

Etiquetas (Tags)

Las etiquetas HTML son los componentes básicos de las páginas web. Cada etiqueta proporciona una indicación al navegador sobre cómo debe presentarse o comportarse un elemento específico en la página.

Elementos Principales: <html>, <head>, <body>, <title>

  • <html>: La etiqueta raíz que encierra todo el contenido HTML.
  • <head>: Contiene metadatos sobre el documento, como su título (<title>) y enlaces a hojas de estilo CSS.
  • <body>: Contiene el contenido visible de la página web.

Ejemplo básico de estructura HTML:

<!DOCTYPE html>
<html>
<head>
  <title>Mi Primera Página</title>
</head>
<body>
  <h1>Bienvenidos a mi sitio web</h1>
  <p>Este es un párrafo de ejemplo.</p>
</body>
</html>

Encabezados: <h1>, <h2>, <h3>, ...

HTML tiene seis niveles de encabezados que se utilizan para definir la jerarquía del contenido, desde el más importante (<h1>) hasta el menos importante (<h6>).

<h1>Título Principal</h1>
<h2>Subtítulo</h2>
<h3>Sección</h3>

Párrafos y Texto en Negrita: <p>, <b>

  • <p>: Define un párrafo.
  • <b>: Hace que el texto aparezca en negrita.
<p>Este es un párrafo. <b>Este texto está en negrita.</b></p>

Saltos de Línea y Reglas Horizontales: <br>, <hr>

  • <br>: Añade un salto de línea.
  • <hr>: Añade una línea horizontal para separar secciones.
<p>Línea 1<br>Línea 2</p>
<hr>
<p>Nueva sección</p>

Listas: Ordenadas, Desordenadas y de Definición

  • Listas Ordenadas (<ol>): Los elementos de la lista están numerados.
  • Listas Desordenadas (<ul>): Los elementos de la lista están marcados con viñetas.
  • Listas de Definición (<dl>): Contienen términos y sus definiciones.
<ol>
  <li>Primer elemento</li>
  <li>Segundo elemento</li>
</ol>

<ul>
  <li>Elemento de lista</li>
  <li>Otro elemento</li>
</ul>

<dl>
  <dt>Término 1</dt>
  <dd>Definición 1</dd>
  <dt>Término 2</dt>
  <dd>Definición 2</dd>
</dl>

Enlaces: <a>

Los enlaces permiten la navegación entre páginas. Se crean utilizando la etiqueta <a> y el atributo `href`.

Ejemplo de enlace absoluto:

<a href="https://www.example.com">Visita Ejemplo</a>

Ejemplo de enlace relativo:

<a href="/pagina-interna.html">Página Interna</a>

Identificadores y Anclas: `id` y enlaces internos

Para enlazar a una parte específica de una página, se usa el atributo `id` en el elemento destino.

<h1 id="titulo1">Título 1</h1>
<a href="#titulo1">Ir a Título 1</a>

Imágenes: <img>

Las imágenes se insertan usando la etiqueta <img> y los atributos `src`, `alt`, `width` y `height`.

<img src="imagen.jpg" alt="Descripción de la imagen" width="200" height="100">

CSS: Diseño y Estilo de Páginas Web

Introducción a CSS

CSS (Cascading Style Sheets) se utiliza para controlar la presentación de los elementos HTML. Permite definir estilos para elementos específicos o aplicar reglas a múltiples elementos de manera consistente.

Concepto de Caja en CSS

Cada elemento HTML se puede imaginar como una caja que contiene contenido y tiene propiedades de presentación como margen, borde, relleno y contenido.

Uso de CSS Externo

Para separar la estructura (HTML) de la presentación (CSS), se utilizan archivos CSS externos, enlazados en el <head> del documento HTML.

<head>
  <link rel="stylesheet" type="text/css" href="styles.css">
</head>

Propiedades CSS

Algunas propiedades comunes de CSS incluyen:

  • display: Define cómo se debe mostrar un elemento (`block`, `inline`, `flex`).
  • flex-direction: Establece la dirección del eje principal (`row`, `column`).
  • justify-content: Alinea los elementos a lo largo del eje principal (`center`, `flex-start`, `flex-end`).
  • align-items: Alinea los elementos a lo largo del eje transversal (`center`, `stretch`).
  • font-family: Define la fuente del texto.
  • text-align: Alinea el texto (`left`, `right`, `center`).
  • padding: Define el espacio interior alrededor del contenido de un elemento.
body {
  display: flex;
  flex-direction: column;
  justify-content: center;
  align-items: center;
  font-family: 'Source Serif Pro', serif;
  text-align: center;
  padding: 5px 20% 0 20%;
}

Estructura de Programas en JavaScript

JavaScript es un lenguaje de programación fundamental para el desarrollo web, y entender su estructura es esencial para crear aplicaciones eficaces. Esta presentación aborda conceptos clave como bindings (variables), flujo de control y bucles (loops).

Bindings o Variables

¿Qué es un Binding?

Un binding (o variable) en JavaScript es una forma de almacenar y manipular datos. Permite que un programa recuerde valores y estados internos. Los bindings se crean usando palabras clave como `let`, `const` y `var`. Aquí nos centraremos en `let`.

Ejemplo básico:

let op = 5 * 5;

En este caso, `let` indica que estamos definiendo un binding llamado `op` y le asignamos el valor `25`.

Uso y Actualización de Bindings

Una vez definido un binding, podemos usar su nombre para referirnos al valor que almacena. Los bindings no están vinculados permanentemente a un valor y se pueden actualizar.

Ejemplo:

let ten = 10;
console.log(ten * ten);  // 100

Actualizar un binding:

let mood = "light";
console.log(mood);  // "light"

mood = "dark";
console.log(mood);  // "dark"

Ejemplo de actualización basada en un cálculo:

let luigisDebt = 140;
luigisDebt = luigisDebt - 35;
console.log(luigisDebt);  // 105

Flujo de Control

Cuando un programa tiene múltiples declaraciones, estas se ejecutan secuencialmente, de arriba hacia abajo, similar a leer una historia.

Ejecución Condicional

A veces, necesitamos que el programa tome decisiones basadas en ciertas condiciones. Esto se logra con la declaración `if`.

Ejemplo:

let number = prompt("Enter a number:");
if (number > 0) {
  console.log("The number is positive.");
}

Bucles (Loops)

Los bucles permiten repetir un bloque de código múltiples veces. Esto es útil para realizar tareas repetitivas sin tener que escribir el mismo código varias veces.

While Loops

Un `while` loop repite un bloque de código mientras una condición sea verdadera.

Ejemplo de imprimir números pares de 0 a 12:

let number = 0;
while (number <= 12) {
  console.log(number);
  number = number + 2;
}

For Loops

Un `for` loop es una forma más compacta de repetir código. Es útil cuando sabemos de antemano cuántas veces queremos repetir el bloque.

Ejemplo de calcular \(2^{10}\):

let result = 1;
for (let counter = 0; counter < 10; counter = counter + 1) {
  result = result * 2;
}
console.log(result);  // 1024

Switch

La declaración `switch` se utiliza para evaluar una expresión y ejecutar diferentes bloques de código basados en el valor de la expresión.

Ejemplo:

let day = "Monday";
switch (day) {
  case "Monday":
    console.log("Start of the work week");
    break;
  case "Wednesday":
    console.log("Midweek");
    break;
  case "Friday":
    console.log("End of the work week");
    break;
  default:
    console.log("Another day");
    break;
}

React, Hooks, y JSX

React es una biblioteca de JavaScript para construir interfaces de usuario tanto en aplicaciones web como en móviles. Se basa en componentes, que son piezas reutilizables de código que representan partes de la interfaz de usuario. En esta presentación, cubriremos los conceptos clave de React, incluidos los hooks, JSX, estructuras de datos como objetos y arreglos, y métodos.

Estructuras de Datos: Objetos y Arreglos

Arreglos (Arrays)

Los números, booleanos y cadenas de texto son los componentes básicos de las estructuras de datos. Para trabajar con conjuntos de datos, los arreglos (arrays) y objetos son fundamentales.

Ejemplo de arreglo:

let listOfNumbers = [2, 3, 5, 7, 11];

console.log(listOfNumbers[2]);  // 5
console.log(listOfNumbers[0]);  // 2
console.log(listOfNumbers[2 - 1]);  // 3

Métodos de Arreglos

Los arreglos en JavaScript tienen varios métodos útiles que permiten manipular y trabajar con los datos de manera eficiente.

Ejemplo de métodos:

let sequence = [1, 2, 3];
sequence.push(4);  // Añade 4 al final
sequence.push(5);  // Añade 5 al final

console.log(sequence);  // [1, 2, 3, 4, 5]
console.log(sequence.pop());  // Elimina y devuelve el último elemento (5)
console.log(sequence);  // [1, 2, 3, 4]

Objetos (Objects)

Los objetos son colecciones de pares clave-valor y permiten almacenar datos más complejos.

Ejemplo de objeto:

let anObject = { left: 1, right: 2 };
console.log(anObject.left);  // 1

delete anObject.left;
console.log(anObject.left);  // undefined

console.log("left" in anObject);  // false
console.log("right" in anObject);  // true

Para obtener las claves de un objeto:

console.log(Object.keys({ x: 0, y: 0, z: 2 }));  // ["x", "y", "z"]

React

React permite construir interfaces de usuario mediante componentes individuales que se pueden combinar en pantallas, páginas y aplicaciones completas.

Componentes

Los componentes en React son funciones de JavaScript que pueden recibir propiedades (`props`) y devolver elementos de React que describen lo que se debe mostrar en la pantalla.

Ejemplo de componente:

function Video({ video }) {
  return (
    <div>
      <Thumbnail video={video} />
      <a href={video.url}>
        <h3>{video.title}</h3>
        <p>{video.description}</p>
      </a>
      <LikeButton video={video} />
    </div>
  );
}

JSX

JSX es una extensión de la sintaxis de JavaScript que permite escribir elementos de React utilizando una sintaxis similar a HTML. Se traduce a llamadas de funciones de JavaScript que crean elementos de React.

Ejemplo de JSX:

const element = <h1>Hello, world!</h1>;

Listas y Condicionales

Para mostrar contenido condicionalmente, se puede usar una declaración `if`. Para mostrar listas, se puede usar el método `map` de los arreglos.

Ejemplo de componente que muestra una lista:

function VideoList({ videos, emptyHeading }) {
  const count = videos.length;
  let heading = emptyHeading;
  if (count > 0) {
    const noun = count > 1 ? 'Videos' : 'Video';
    heading = count + ' ' + noun;
  }
  return (
    <section>
      <h2>{heading}</h2>
      {videos.map(video =>
        <Video key={video.id} video={video} />
      )}
    </section>
  );
}

Hooks

Los hooks permiten utilizar características de React, como el estado y los efectos, dentro de componentes funcionales. Hay hooks integrados que se pueden usar y también se pueden crear hooks personalizados.

Hook de Estado: `useState`

El hook `useState` permite añadir estado a un componente funcional. Proporciona una variable de estado y una función para actualizarla.

Ejemplo de `useState`:

const [index, setIndex] = useState(0);

function handleClick() {
  setIndex(index + 1);
}

Hook de Efecto: `useEffect`

El hook `useEffect` permite realizar efectos secundarios en los componentes, como conectar con APIs externas, suscribirse a eventos, o manipular el DOM.

Ejemplo de `useEffect`:

import { useEffect, useState } from 'react';
import { createConnection } from './chat.js';

function ChatRoom({ roomId }) {
  const [serverUrl, setServerUrl] = useState('https://localhost:1234');

  useEffect(() => {
    const connection = createConnection(serverUrl, roomId);
    connection.connect();
    return () => {
      connection.disconnect();
    };
  }, [serverUrl, roomId]);
}

Rutas, Fetch, Métodos HTTP y Anatomía de URLs

En esta presentación, exploraremos cómo manejar rutas, realizar solicitudes HTTP utilizando la API Fetch, y los diferentes métodos HTTP (GET, POST, PUT, DELETE, etc.) en el contexto del desarrollo web con React. También desglosaremos la anatomía de una URL para entender mejor su estructura y componentes.

Fetch

Fetch es una API basada en promesas que permite realizar solicitudes HTTP asíncronas en JavaScript. Es una alternativa mejorada que se integra con conceptos avanzados de HTTP, como CORS (Cross-Origin Resource Sharing).

Uso Básico de Fetch

Fetch se usa para hacer solicitudes HTTP y manejar las respuestas de manera asíncrona. Aquí hay un ejemplo básico:

async function logMovies() {
  const response = await fetch("http://example.com/movies.json");
  const movies = await response.json();
  console.log(movies);
}

En este ejemplo, `fetch` hace una solicitud GET a la URL proporcionada, y el método `response.json()` se usa para convertir la respuesta en un objeto JavaScript.

Opciones de Solicitud

El método `fetch` puede aceptar un segundo parámetro opcional, un objeto de configuración (init), que permite controlar diversas configuraciones de la solicitud.

Ejemplo de uso de `POST`:

async function postData(url = "", data = {}) {
  const response = await fetch(url, {
    method: "POST", // *GET, POST, PUT, DELETE, etc.
    headers: {
      "Content-Type": "application/json",
    },
    body: JSON.stringify(data), // Los datos deben coincidir con el tipo de contenido
  });
  return response.json(); // Convierte la respuesta en un objeto JavaScript
}

Métodos HTTP

Cada método HTTP tiene un propósito específico en la interacción con un servidor:

GET

El método GET se utiliza para recuperar datos del servidor. No incluye un cuerpo de solicitud.

Ejemplo:

fetch('/products')
  .then(response => response.json())
  .then(data => console.log(data));

POST

El método POST se utiliza para crear nuevos recursos. Incluye un cuerpo de solicitud con los datos del nuevo recurso.

Ejemplo:

fetch('/products', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'New Product',
    price: 100
  }),
})
.then(response => response.json())
.then(data => console.log(data));

PUT

El método PUT se utiliza para reemplazar un recurso existente con una nueva versión.

Ejemplo:

fetch('/products/123', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    name: 'Updated Product',
    price: 120
  }),
})
.then(response => response.json())
.then(data => console.log(data));

PATCH

El método PATCH se utiliza para actualizar propiedades específicas de un recurso sin sobrescribirlo por completo.

Ejemplo:

fetch('/products/123', {
  method: 'PATCH',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    price: 130
  }),
})
.then(response => response.json())
.then(data => console.log(data));

DELETE

El método DELETE se utiliza para eliminar un recurso del servidor.

Ejemplo:

fetch('/products/123', {
  method: 'DELETE',
})
.then(response => response.json())
.then(data => console.log(data));

Anatomía de URLs

Una URL (Uniform Resource Locator) es la dirección que utilizamos para acceder a recursos en la web. Comprender su estructura es esencial para construir rutas y manejar solicitudes HTTP.

Estructura de una URL

Una URL típica se compone de varios componentes:

https://www.example.com:8080/path/to/resource?query=param#fragment
  • Protocolo (`https://`): Indica el protocolo utilizado (HTTP, HTTPS, FTP, etc.).
  • Nombre del dominio (`www.example.com`): El nombre del servidor donde reside el recurso.
  • Puerto (`:8080`): El puerto del servidor (opcional, por defecto es 80 para HTTP y 443 para HTTPS).
  • Ruta (`/path/to/resource`): La ruta al recurso específico en el servidor.
  • Consulta (`?query=param`): Una cadena de consulta que envía datos adicionales al servidor.
  • Fragmento (`#fragment`): Una referencia a una parte específica del recurso.

Rutas en React

React Router es una biblioteca estándar para manejar el enrutamiento en aplicaciones React. Permite definir múltiples rutas y cómo se deben renderizar los componentes para cada ruta.

Ejemplo básico de configuración de rutas:

import { BrowserRouter as Router, Route, Switch } from 'react-router-dom';

function App() {
  return (
    <Router>
      <Switch>
        <Route exact path="/" component={Home} />
        <Route path="/about" component={About} />
        <Route path="/contact" component={Contact} />
      </Switch>
    </Router>
  );
}

Node y Express

Node.js y Express son herramientas poderosas para desarrollar aplicaciones del lado del servidor usando JavaScript. Esta presentación cubrirá los conceptos básicos de Node.js y Express, sus ventajas, y cómo se utilizan juntos para construir aplicaciones web eficientes y escalables.

Node.js

¿Qué es Node.js?

Node.js es un entorno de ejecución de JavaScript de código abierto y multiplataforma que permite a los desarrolladores crear aplicaciones y herramientas del lado del servidor. Node.js utiliza el motor V8 de Google, que compila JavaScript a código máquina, proporcionando un alto rendimiento.

Ventajas de Node.js

  • Gran rendimiento: Node.js está diseñado para optimizar el rendimiento y la escalabilidad, especialmente en aplicaciones web.
  • JavaScript unificado: Al usar JavaScript tanto en el servidor como en el cliente, se reduce el tiempo necesario para cambiar entre lenguajes.
  • Gestor de paquetes (NPM): Node.js incluye NPM, que proporciona acceso a miles de paquetes reutilizables.
  • Comunidad activa: Hay una gran comunidad de desarrolladores que contribuyen al ecosistema de Node.js.

Limitaciones de Node.js

Node.js no incluye soporte directo para muchas tareas comunes de desarrollo web, como manejar diferentes verbos HTTP, gestionar rutas o servir archivos estáticos. Para estas tareas, se utilizan frameworks como Express.

Express.js

¿Qué es Express.js?

Express.js es el framework web más popular para Node.js. Simplifica el desarrollo de aplicaciones web y APIs al proporcionar una serie de características robustas para gestionar peticiones HTTP, rutas y middleware.

Características de Express.js

  • Manejo de peticiones HTTP: Express permite escribir manejadores de peticiones para diferentes verbos HTTP (GET, POST, DELETE, etc.) en diferentes rutas URL.
  • Renderización de vistas: Integra con motores de plantillas para generar respuestas dinámicas.
  • Configuración de aplicaciones: Permite establecer ajustes como el puerto de conexión y la localización de las plantillas.
  • Middleware: Añade procesamiento adicional en cualquier punto de la cadena de manejo de peticiones.

Ejemplo Básico de Express

Instalar Express:

npm install express

Crear una aplicación básica:

const express = require('express');
const app = express();

app.get('/', (req, res) => {
  res.send('Hello, world!');
});

app.listen(3000, () => {
  console.log('Server is running on port 3000');
});

En este ejemplo, `app.get` define una ruta para manejar las peticiones GET a la URL raíz (`/`), y `res.send` envía una respuesta al cliente. `app.listen` inicia el servidor en el puerto 3000.

Node.js y Express en Aplicaciones Dinámicas

En aplicaciones web dinámicas, el servidor espera recibir peticiones HTTP del navegador o cliente. Al recibir una petición, la aplicación determina la acción adecuada basándose en la estructura de la URL y la información en la petición (métodos POST o GET).

Ejemplo de Aplicación Dinámica

Supongamos que queremos manejar peticiones para un blog con la capacidad de crear y listar publicaciones.

const express = require('express');
const app = express();
const bodyParser = require('body-parser');

app.use(bodyParser.json());

let posts = [];

// Ruta para listar publicaciones
app.get('/posts', (req, res) => {
  res.json(posts);
});

// Ruta para crear una nueva publicación
app.post('/posts', (req, res) => {
  const post = req.body;
  posts.push(post);
  res.status(201).json(post);
});

app.listen(3000, () => {
  console.log('Server is running on port 3000');
});

Este ejemplo usa `body-parser` para parsear el cuerpo de las peticiones entrantes como JSON. La ruta `GET /posts` devuelve todas las publicaciones, mientras que la ruta `POST /posts` permite crear una nueva publicación.

Node, Express y PostgreSQL

Node.js, Express y PostgreSQL son herramientas esenciales para el desarrollo de aplicaciones web del lado del servidor. Esta presentación cubre cómo integrar estas tecnologías para crear aplicaciones web eficientes y escalables.

PostgreSQL

¿Qué es PostgreSQL?

PostgreSQL, comúnmente conocido como Postgres, es un sistema de gestión de bases de datos relacional y de código abierto. Compite con otros sistemas de bases de datos como MySQL, Microsoft SQL Server y MariaDB.

Características de PostgreSQL

  • Código abierto y gratuito.
  • Alta conformidad con el estándar SQL.
  • Extensibilidad: Permite añadir nuevos tipos de datos, operadores y funciones.
  • Soporte para transacciones ACID.
  • Sistema robusto para la gestión de datos estructurados y no estructurados.

Node-postgres (pg)

¿Qué es node-postgres?

Node-postgres, o `pg`, es un cliente PostgreSQL no bloqueante para Node.js. Incluye una colección de módulos de Node.js para interactuar con una base de datos PostgreSQL.

Características de node-postgres

  • Soporte para callbacks, promesas y async/await.
  • Pooling de conexiones.
  • Sentencias preparadas.
  • Cursors y análisis de tipos ricos.
  • Enlaces C/C++.

Uso del terminal de PostgreSQL (psql)

El terminal interactivo `psql` permite interactuar con PostgreSQL desde la línea de comandos.

Comandos comunes de psql

  • \q: Salir de la conexión `psql`.
  • \c: Conectarse a una nueva base de datos.
  • \dt: Listar todas las tablas.
  • \du: Listar todos los roles.
  • \list: Listar todas las bases de datos.

Ejemplo de conexión:

psql -d postgres -U catedratec

Crear un rol en PostgreSQL

CREATE ROLE catedratec WITH LOGIN PASSWORD 'password';
ALTER ROLE catedratec CREATEDB;

Crear una base de datos

CREATE DATABASE catedraapi;

Crear una tabla

CREATE TABLE users (
  ID SERIAL PRIMARY KEY,
  name VARCHAR(30),
  email VARCHAR(30)
);

Insertar datos en la tabla

INSERT INTO users (name, email)
  VALUES ('David', 'david@gmail.com'), ('Frida', 'frida@gmail.com');

Configuración de un servidor Express

Instalación de dependencias

Primero, instalamos Express y los módulos necesarios:

npm install express body-parser pg

Configuración básica del servidor Express

Creamos un archivo `server.js` y configuramos el servidor Express:

const express = require('express')
const bodyParser = require('body-parser')
const app = express()
const port = 3000

app.use(bodyParser.json())
app.use(bodyParser.urlencoded({ extended: true }))

app.listen(port, () => {
  console.log(`App running on port ${port}.`)
})

Conexión a PostgreSQL con node-postgres

Configuramos la conexión a PostgreSQL:

const Pool = require('pg').Pool
const pool = new Pool({
  user: 'catedratec',
  host: 'localhost',
  database: 'catedraapi',
  password: 'password',
  port: 5432,
})

Rutas y controladores en Express

Definimos rutas para obtener todos los usuarios y obtener un usuario por ID:

const getUsers = (request, response) => {
  pool.query('SELECT * FROM users ORDER BY id ASC', (error, results) => {
    if (error) {
      throw error
    }
    response.status(200).json(results.rows)
  })
}

const getUserById = (request, response) => {
  const id = parseInt(request.params.id)

  pool.query('SELECT * FROM users WHERE id = $1', [id], (error, results) => {
    if (error) {
      throw error
    }
    response.status(200).json(results.rows)
  })
}

app.get('/users', getUsers)
app.get('/users/:id', getUserById)
